基础类型

null

const data: null = null;，null类型的变量只能被赋值为null，所以通常不使用。

undefined

const data: undefined = undefined;，undefined类型的变量只能被赋值为undefined，所以通常不使用。

void

this type

JavaScript遵循静态的函数作用域（es6后有了块级作用域），即变量的作用域可以通过静态的代码结构直接推断得出，无需在运行时动态确定。唯一的例外是this变量，this变量的指向存在4种不同的情况，必须在运行时才能确定最终指向的变量，这4种不同的情况分别是：

  1. 直接调用函数时，this指向window或undefined（严格模式下）。
  2. 作为对象的成员方法调用时，this指向对象。
  3. 作为构造函数调用时，this指向新生成的空对象。
  4. 调用函数时使用了call，apply或bind方法时，指向传入的参数。

这导致this最终指向的对象极不可控，是js error的高发地。为了避免因this而导致的js error，JavaScript中通常的做法是在使用this对象的方法前，先做检查，例如：

shape

描述对象类型的类型标注称为shape，如上例中的：

shape是typescript对对象类型变量进行静态检查的依据，比较对象每个属性的命名与值的类型是否符合shape中的定义来确定程序中是否出现错误。可以直接书写shape作为标注来给变量定义类型，也可以给shape命名，方便复用shape，定义interface的过程即为shape命名的过程，命名后的shape可以直接使用shape名进行类型标注。

interface

定义interface的过程即为shape命名的过程。

interface方便我们复用shape，还提供了更多的功能，例如支持更多的修饰符，支持泛型，支持继承等。

何时使用shape进行标注？

  • 不会在多处被使用的shape，例如模块的内部函数（不会被export的函数），类的私有函数的参数列表，返回值的标注。
  • 简单的shape，例如事件处理函数参数列表的标注。

修饰符

typescript中class支持四种修饰符：

  • readonly。只能应用在成员变量，保证该成员变量不可变，该修饰符前还可使用其他3个修饰符。
  • private。与传统面向对象语言中的功能相同。
  • protected。与传统面向对象语言中的功能相同。
  • public。与传统面向对象语言中的功能相同。

静态部分，实例部分

typescript中将class分割为实例部分与静态部分。
类的实例部分包含类的成员变量与成员方法，标注变量a为类A的实例的方法为：let a: A。
typescript中类的静态部分包括类的静态方法以及类的构造函数，标注变量a为类A本身的方法为：let a: typeof A。

实现interface

类可以声明自身实现了某个interface，这在先有interface再实现类的场景下十分有用。当类的成员变量，成员方法不符合interface时，typescript会报错。

需要注意，interface中不支持private，protected，public修饰符。

import，export

typescript中，类型定义也可以被导入/导出，语法和导入/导出普通对象相同。

泛型

泛型是程序设计语言的一种风格或范式。泛型允许程序员在强类型语言中编写代码时使用一些以后才指定的类型，在实例化时作为参数指明这些类型。

简单的说，泛型提供了在程序中延迟指定类型的能力，增加了程序的灵活性，拓展性。

typescript在多种场景下都提供了泛型的支持。
